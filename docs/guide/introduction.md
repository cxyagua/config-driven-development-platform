# 简介

**CDDP**（Config-Driven Development Platform，配置驱动开发平台）是一个以「配置」为中心，覆盖开发、预览与执行全流程的可视化平台。

## 它解决什么问题

在传统开发模式下，业务逻辑往往散落在代码中，调整一个行为就需要修改代码、重新编译、重新部署。CDDP 将可变的部分抽象为「配置」：

- **开发**：在工作台中可视化地编写与组织配置文件
- **预览**：配置改动可即时预览，所见即所得
- **执行**：服务端解析并校验配置，按配置驱动业务执行

## 核心模块

| 模块 | 说明 | 技术栈 |
| --- | --- | --- |
| `workbench` | 工作台前端，配置管理与即时预览 | Vue 3 + TypeScript + Vite |
| `server` | 服务端，配置解析、验证与执行 | Python + FastAPI |

> 后续还将规划 `skills` 技能库、`reference` 参考、`components` 组件库、`playground` 实验平台等模块。

## 适用场景

- 需要频繁调整业务参数、规则、流程的应用
- 多环境（dev / uat / prod）配置需要隔离与同步的团队
- 希望把「配置」从代码中解耦，降低发布成本的工程项目
